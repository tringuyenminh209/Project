■ 1. 本書の目的

単体試験（15_単体試験仕様書）はSQLite in-memoryで業務ルールを高速に検証した。バックエンドが外部APIを一切呼び出さない本システムでは、単体試験がfakeで代替した境界は「実MySQL」のみに限定される — PMSのようなS3・Reverbのfake置換は存在しない。代わりに本書はフロントエンドが実際にGoogle Maps/Geocoding APIを呼び出す境界を重点的に扱う。特にIT-MAP-005（位置情報未確定リストの実描画確認）は、12_詳細設計書v1.1で修正した重大リスクをAPI応答レベルだけでなく実際のブラウザ画面で再確認する、本書で最も重要な試験である。

■ 13. まとめ

本書は「単体で正しいものが、実物の境界をまたいでも正しいか」を確認する。本システムはPMSと異なりS3・WebSocketの実物結合試験を持たないため、実質的な重心は2つ — ①実MySQLでのatomic upsert・JSON列・bulk insertの実挙動（6章）、②フロントエンドが実際にGoogle Maps/Geocoding APIと通信する境界（7章）である。特にIT-MAP-005は、12_詳細設計書v1.1で発見・修正した「位置情報のない報告の誤った地図表示」というリスクを、API応答のJSON構造だけでなく実際にブラウザで目視確認する試験であり、本システムの試験計画全体の中で最も優先度が高い。IT-ENV-001（try_filesディープリンク）は本番ビルド配信でしか検証できないため、dev serverの動作確認で代用しないこと。
