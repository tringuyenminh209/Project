# Disaster Safety Report System — 設計文書一覧

Portfolio Roadmap（SE/BrSE 2026-2027）Project 04。HR & Attendance System（Project 01）・EC Site（Project 02）・Project Management System（Project 03）と同じ21文書構成（DSR-00〜DSR-20）。

**現在のステータス: `00`〜`20`の全21文書、本文完成（2026/07/23）。設計フェーズ（DSR-00〜20）を完了した。**

`00_開発計画書`で以下の3論点を確定済み（詳細は同文書の該当章を参照）:

1. **Tech Stack**: Google Maps API に決定（Mapbox不採用）— 00_開発計画書 6章
2. **`companies`の定義**: 単一企業内システム（マルチテナントSaaSではない）に決定 — 00_開発計画書 3章・9章
3. **ユーザーロール**: 一般社員 / 部門管理者 / Admin の3層に決定 — 00_開発計画書 3章

`02_要件定義書`でさらに以下を確定済み（00が08/09へ先送りしていた論点）:

4. **安否ステータスの選択肢**: `safe`（安全）/ `needs_help`（要支援）の2値。「未確認」は未報告の派生表示 — BR-RPT-001
5. **再報告の方式**: `UNIQUE(disaster_id, employee_id)`のUPDATE方式、履歴は残さない — BR-RPT-002
6. **位置情報の持ち方**: `locations`はマスタ参照、`safety_reports`は報告時点のスナップショット — BR-RPT-003

以降の全文書（`03_ユースケース`〜）はこれらを前提として執筆する。
