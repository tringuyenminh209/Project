■ 1. 本書の目的

本書はEC SiteをDevelopment環境からProduction環境（AWS EC2 + Docker + RDS + S3）へリリースする手順を定義する。Project 01の手順をベースに、Stripe本番キー切替・S3設定の追加手順を含める。

■ 2. リリース方針
方針ID方針内容
REL-POL-001段階的リリースLocal → Development → Production の順で確認する。
REL-POL-002Migration先行確認本番Migration前に、Development環境で同一Migrationを実行し失敗しないことを確認する。
REL-POL-003決済は最後に本番化Stripeのテストキーから本番キーへの切替は、他の全確認が完了した最終段階でのみ行う。

■ 5. リリース手順
1. mainブランチの最新コミットを確認する
2. Production EC2にSSH接続する
3. git pullで最新コードを取得する
4. docker compose build（変更がある場合）
5. docker compose up -d
6. php artisan migrate --force（6章参照）
7. php artisan config:cache / route:cache
8. 8章のSmoke Testを実施する
■ 6. Migration手順
1. RDSの手動スナップショットを取得する（念のためのバックアップ）
2. Development環境で同一Migrationが成功することを事前確認済みであることを再確認する
3. php artisan migrate --force を実行する
4. php artisan migrate:status で全Migrationが実行済みであることを確認する

16テーブルすべてに外部キー制約・CHECK制約があるため、Migration順序は09_テーブル定義3章のテーブル一覧順（依存関係の浅い順: users→addresses→categories→products→...）を厳守する。

■ 7. Stripe設定切替手順
1. Stripeダッシュボードで本番用APIキー（sk_live_...）を発行する
2. .envのSTRIPE_KEY/STRIPE_SECRETを本番用に差し替える
3. Stripeダッシュボードで本番Webhookエンドポイント（https://example.com/api/webhooks/stripe）を登録する
4. 発行されたWebhook Signing SecretをSTRIPE_WEBHOOK_SECRETに設定する
5. 少額の実カードでテスト決済を1件行い、Webhookが正しく処理されることを確認する（初回のみ）
■ 8. リリース後確認（Smoke Test）
□ /api/health が200を返す
□ 商品一覧・詳細が表示される
□ テスト会員でログインできる
□ カート追加〜注文確定〜決済（テストカードまたは少額実カード）が成功する
□ Admin管理画面にログインできる
■ 9. ロールバック手順
1. docker compose down
2. 直前の安定版タグへgit checkout
3. docker compose up -d
4. Migrationをロールバックしたテーブルがある場合、RDSスナップショットから復元を検討する
   （注文・決済データを含むため、Migrationのロールバックは原則as new columnの追加のみに限定し、
    破壊的変更は避ける方針とする）

■ 12. まとめ

本プロジェクト特有のリリースリスクはStripe本番キー切替（7章）である。テストキーのまま本番公開してしまう事故を防ぐため、4章チェックリストで必ず確認する。
