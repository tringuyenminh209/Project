Chương 02 — Danh mục & Sản phẩm (Catalog)

Chương này dựng toàn bộ "xương sống dữ liệu" của catalog: 5 bảng categories / products / product_variants / product_images / inventories (TBL-003〜007), model + relation Eloquent, và 2 API công khai quan trọng nhất — danh sách sản phẩm (API-006) với giá trị tổng hợp price_min/price_max, và chi tiết sản phẩm (API-007) kèm tồn kho theo variant. Phía Admin, làm mẫu đăng ký variant (API-025) vì nó chứa pattern quan trọng: transaction tạo đồng thời product_variants + inventories.

⚠️ Điều kiện vào chương

Đã hoàn thành Chương 01, bao gồm cả bài tập Middleware EnsureRole — các route /admin/* trong chương này dùng alias role:admin (12_詳細設計書 8章). Nếu chưa làm, quay lại 課題 số 4 của Chương 01 trước.

💡 Vì sao price_min/price_max không lưu trong bảng products?

Giá thuộc về variant (SKU), không thuộc về product — 10_API設計.md 6.6節 ghi rõ đây là "集計値" (giá trị tổng hợp MIN/MAX từ product_variants.price, không tồn tại trong bảng). Nếu lưu trùng vào products, mỗi lần sửa giá variant sẽ phải đồng bộ 2 nơi — vi phạm single source of truth. Đổi lại, mỗi query danh sách phải aggregate; Eloquent withMin/withMax sinh subquery gọn và đủ nhanh ở quy mô NFR-002 (100 user đồng thời). Khi cần tối ưu thêm mới tính đến Redis cache (NFR-011, bonus).

💡 Vì sao đăng ký variant phải tạo luôn bản ghi inventories rỗng trong cùng transaction?

BR-INV-001: tồn kho quản lý tập trung tại inventories, mỗi variant đúng 1 dòng (UNIQUE variant_id). Dòng này còn là đối tượng khoá của SELECT ... FOR UPDATE ở nghiệp vụ đặt hàng (BR-INV-007) — nếu variant tồn tại mà dòng inventory chưa có, luồng đặt hàng sau này sẽ không có gì để khoá và phải xử lý case "NULL inventory" ở khắp nơi. Vì vậy REQ-020 quy định tạo variant thì tự động tạo inventory với quantity_available=0 (12_詳細設計書 5.12節), và DD-POL-002 buộc 2 insert này nằm trong 1 DB::transaction().

💡 Vì sao "không tìm thấy" và "inactive" đều trả E007?

TBL-POL-003: catalog dùng logical delete (đổi status) thay vì xoá vật lý — vì order_items tham chiếu variant bằng FK RESTRICT, xoá thật sẽ phá lịch sử đơn hàng. Với Customer, một sản phẩm đã inactive tương đương "không tồn tại" (API-007: cả 2 case đều E007) — không cần tiết lộ sự khác biệt. Còn Admin thì ngược lại: API-024 trả cả inactive (để quản lý), đây là điểm khác nhau giữa API công khai và API admin cùng đọc một bảng.

Migration 5 bảng catalog（TBL-003〜007）

Tạo theo đúng thứ tự phụ thuộc FK: categories → products → product_variants → product_images → inventories. Chú ý 3 điểm dễ sai: (1) inventories không có created_at (ngoại lệ TBL-POL-004); (2) UNIQUE 3 cột (product_id, size, color) trên variants; (3) CHECK constraint chống tồn kho âm phải thêm bằng DB::statement vì Blueprint của Laravel không có API cho CHECK.

Exception E007 — ResourceNotFoundException

Tên class lấy đúng theo 12_詳細設計書.md 11章: ResourceNotFoundException, kế thừa ModelNotFoundException chuẩn của Laravel (chủ ý của thiết kế: sau này exception từ findOrFail() cũng có thể quy về cùng một chỗ xử lý). Vì không kế thừa ApiException của Chương 01, class này tự mang render() riêng trả đúng envelope 3.3節.

app/Exceptions/ResourceNotFoundException.phpphp
<?php
// app/Exceptions/ResourceNotFoundException.php
// 10_API設計.md 7章 E007: 対象データ未検出（404）
// 12_詳細設計書.md 11章の指定どおりLaravel標準のModelNotFoundExceptionを継承する
// 「存在しない」と「status=inactive」を同じE007に丸める（API-007の仕様どおり。
//  inactiveであることをCustomerに区別して見せる必要はない）

namespace App\Exceptions;

use Illuminate\Database\Eloquent\ModelNotFoundException;
use Illuminate\Http\JsonResponse;

class ResourceNotFoundException extends ModelNotFoundException
{
    public function __construct(string $message = '対象のデータが見つかりません。')
    {
        parent::__construct($message);
    }

    public function render(): JsonResponse
    {
        return response()->json([
            'success' => false,
            'error' => [
                'code' => 'E007',
                'message' => $this->getMessage(),
                'details' => [],
            ],
        ], 404);
    }
}

API-006 / API-007 — ProductService & ProductController

Lưu ý: 12_詳細設計書.md 6章 gợi ý accessor getPriceRangeAttribute (MIN/MAX của variants) trên model Product. Guide này tính ở tầng query bằng withMin/withMax thay vì accessor — cùng giá trị, nhưng accessor sẽ lazy-load variants của từng sản phẩm trong danh sách (N+1 query), còn withMin/withMax gộp vào 1 subquery cho cả trang. Nếu muốn theo đúng chữ của 詳細設計書, bạn có thể thêm accessor đó cho các chỗ chỉ xử lý 1 sản phẩm.

app/Services/ProductService.phpphp
<?php
// app/Services/ProductService.php
// 10_API設計.md 6.6節(API-006)・6.7節(API-007) / 12_詳細設計書.md 6章

namespace App\Services;

use App\Exceptions\ResourceNotFoundException;
use App\Models\Product;
use Illuminate\Contracts\Pagination\LengthAwarePaginator;

class ProductService
{
    // API-006 商品一覧（検索・絞込・ページング）
    public function list(array $filters): LengthAwarePaginator
    {
        return Product::query()
            ->active() // REQ-005: status=activeのみ（inactiveは表示しない）
            ->with(['category:id,name', 'primaryImage'])
            // price_min/price_maxはテーブルに存在しない集計値（10_API設計 6.6節）
            ->withMin('activeVariants as price_min', 'price')
            ->withMax('activeVariants as price_max', 'price')
            ->when($filters['keyword'] ?? null,
                fn ($q, $kw) => $q->where('name', 'like', "%{$kw}%"))
            ->when($filters['category_id'] ?? null,
                fn ($q, $cid) => $q->where('category_id', $cid))
            ->latest('id')
            // 6.6節のResponse例(total:96, last_page:5)から1ページ20件とする
            ->paginate(20);
    }

    // API-007 商品詳細
    public function detail(int $id): Product
    {
        $product = Product::query()
            ->active()
            ->with([
                'category:id,name',
                // 在庫数はinventoriesからJOINして返す集計値（6.7節）
                'activeVariants.inventory:id,variant_id,quantity_available',
                'images',
            ])
            ->find($id);

        // 存在しない or status=inactive はいずれもE007（6.7節）
        if (!$product) {
            throw new ResourceNotFoundException('対象の商品が見つかりません。');
        }

        return $product;
    }
}

app/Services/VariantService.phpphp
<?php
// app/Services/VariantService.php
// 12_詳細設計書.md 5.12節: VariantService::create（バリエーション登録時の在庫初期化）
// REQ-020受入条件: 「同時にinventoriesの初期レコード(quantity_available=0)が自動作成される」

namespace App\Services;

use App\Exceptions\ResourceNotFoundException;
use App\Models\Product;
use App\Models\ProductVariant;
use Illuminate\Support\Facades\DB;

class VariantService
{
    public function create(int $productId, array $data): ProductVariant
    {
        $product = Product::find($productId);
        if (!$product) {
            throw new ResourceNotFoundException('対象の商品が見つかりません。');
        }

        // DD-POL-002: 複数テーブル(product_variants + inventories)更新は必ず1トランザクション。
        // variantだけ作られてinventoryが無い「片肺」状態を許さない。
        return DB::transaction(function () use ($product, $data) {
            $variant = $product->variants()->create([
                'sku' => $data['sku'],
                'size' => $data['size'] ?? null,
                'color' => $data['color'] ?? null,
                'price' => $data['price'],
                'status' => 'active',
            ]);

            // 在庫の実数は別途API-027（在庫調整）で入荷登録する（5.12節）
            $variant->inventory()->create([
                'quantity_available' => 0,
                'quantity_reserved' => 0,
            ]);

            return $variant->load('inventory');
        });
    }
}

namespace App\Http\Controllers\Admin;

use App\Http\Controllers\Controller;
use App\Http\Requests\Admin\StoreVariantRequest;
use App\Services\VariantService;
use Illuminate\Http\JsonResponse;

class VariantController extends Controller
{
    public function __construct(private VariantService $variantService)
    {
    }

    // POST /admin/products/{id}/variants
    public function store(StoreVariantRequest $request, int $productId): JsonResponse
    {
        $variant = $this->variantService->create($productId, $request->validated());

        return response()->json([
            'success' => true,
            'message' => 'バリエーションを登録しました。',
            'data' => [
                'id' => $variant->id,
                'sku' => $variant->sku,
                'size' => $variant->size,
                'color' => $variant->color,
                'price' => $variant->price,
                'quantity_available' => $variant->inventory->quantity_available,
            ],
        ], 201);
    }
}
