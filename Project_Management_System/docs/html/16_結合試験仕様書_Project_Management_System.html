■ 1. 本書の目的

単体試験（15_単体試験仕様書.md）はSQLite in-memory・Storage::fake・Event::fakeで高速に業務ルールを検証した。本書はそのfakeで代替した境界を実物に置き換えて確認する: 実MySQL（ロック・FK制約）、実S3（presigned URL・オブジェクト実在）、実Reverb（ソケット配信・縮退）、Nginx経由のSPA配信（try_files・プロキシ・wss）。

IT-POL-002fakeの置換15章でfakeにした境界（S3・broadcast・DB）を実物で再確認する。単体で網羅済みの業務ルール分岐は繰り返さない（3章の分担表）。
IT-POL-003縮退確認Reverb停止時のポーリング縮退（NFR-007）と復帰を必ず確認する — bonusが死んでもコアが生きることの実証。

■ 14. まとめ

本書は「単体で正しいものが、実物の境界をまたいでも正しいか」を確認する。要は3つ — ①実MySQLのロック・制約が設計どおり働くか（6章）、②実S3のprivate + presigned 60秒が実際に閉じているか（7章）、③Reverbが死んでもコアが生き続けるか（8章）。加えてIT-ENV-001（try_filesディープリンク）は本番ビルド配信でしか検証できないため、dev serverの動作確認で代用しないこと。IT-SCN-005で発見される「無効化ユーザーの既存トークン」の挙動は、確定後に14_セキュリティ設計.mdへ還元する。
