# 1. 本書の目的

単体試験（`15_単体試験仕様書.md`）はSQLite in-memory・`Storage::fake`・`Event::fake`で高速に業務ルールを検証した。本書はその**fakeで代替した境界を実物に置き換えて**確認する: 実MySQL（ロック・FK制約）、実S3（presigned URL・オブジェクト実在）、実Reverb（ソケット配信・縮退）、Nginx経由のSPA配信（`try_files`・プロキシ・wss）。

# 3. 試験範囲と単体試験との分担

| 境界 | 15（単体・fake） | 16（結合・実物） |

| S3 | Storage::fakeでキー形式・呼び出し | 実バケットへの保存・presigned URLの実取得・期限切れ・削除 |
| Broadcast | Event::fakeで発火有無・afterCommit | 実Reverb経由でブラウザ受信・購読認可・縮退 |

# 14. まとめ

本書は「単体で正しいものが、実物の境界をまたいでも正しいか」を確認する。要は3つ — ①実MySQLのロック・制約が設計どおり働くか（6章）、②実S3のprivate + presigned 60秒が実際に閉じているか（7章）、③Reverbが死んでもコアが生き続けるか（8章）。加えてIT-ENV-001（`try_files`ディープリンク）は本番ビルド配信でしか検証できないため、dev serverの動作確認で代用しないこと。IT-SCN-005で発見される「無効化ユーザーの既存トークン」の挙動は、確定後に`14_セキュリティ設計.md`へ還元する。
