# 1. 本書の目的

本書は、Project Management Systemの全機能をFUNC-IDで一意に採番し、要件（REQ）・ユースケース（UC）・業務フロー（BF）・画面（SCR）との対応を確定する。以降の10_API設計（API-ID⇔FUNC-ID対応）および試験仕様書の基準とする。

# 8. トレーサビリティ

02_要件定義書のREQ-ID → 本書のFUNC-ID → 06_画面設計のSCR-ID →（10_API設計のAPI-ID: 作成後に対応付け）→ 15_単体試験仕様書のUT-IDの順に一意に追跡できる。REQ-001〜028の全件が本書のFUNC-001〜033のいずれかに対応していることを確認済み（REQ-004→FUNC-032、REQ-024→FUNC-029を含む）。

# 9. まとめ

全33機能を9分類に整理した。本システムの特徴は、System実行の機能（FUNC-027〜029, 032〜033）が5件と多いことである — 通知のfan-out・権限判定・ログはユーザーの目に見えないが、12_詳細設計書で最も設計密度が高くなる部分であり、試験仕様書でも独立した試験対象として扱う。また、カンバンの列間移動（FUNC-018）と列内並び替え（FUNC-019）を意図的に別機能として採番した — API上は同一エンドポイントになる可能性があるが、試験観点（ステータス変更の通知非発火 / positionの整合性）が異なるためである。
