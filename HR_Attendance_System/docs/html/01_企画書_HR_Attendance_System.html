■ 1. 企画背景

企業における勤怠管理は、社員の勤務時間、休暇、遅刻、早退などを正確に把握するために重要な業務である。

しかし、小規模な組織や学習用プロジェクトでは、紙、Excel、手入力による管理を想定するケースが多く、入力ミス、確認漏れ、集計作業の負担が発生しやすい。

本企画では、Webアプリケーションとして勤怠管理システムを開発し、社員、管理者、システム管理者がそれぞれ必要な操作を行える環境を提供する。

■ 2. 現状の課題

現状の勤怠管理には、以下の課題がある。

課題内容
打刻情報の管理が煩雑出勤・退勤時刻を手作業で記録すると、入力漏れや記録ミスが発生しやすい。
勤怠状況の確認に時間がかかる管理者が社員ごとの勤務状況を確認する際、集計や検索に時間がかかる。
休暇申請の承認状況が分かりにくい申請、承認、却下の状態を一元管理できない場合、確認漏れが発生する。
月次レポート作成が手作業になる勤務時間や休暇情報を集計する作業に時間がかかる。
権限管理が不十分一般社員、管理者、システム管理者の操作範囲を明確に分ける必要がある。
■ 3. 企画目的

本システムの目的は、勤怠情報をWeb上で一元管理し、社員と管理者の業務効率を向上させることである。

具体的には、以下を実現する。

  • 社員が出勤・退勤を簡単に打刻できる
  • 社員が自分の勤怠履歴を確認できる
  • 社員が休暇申請を行える
  • 管理者が休暇申請を承認・却下できる
  • 管理者が勤怠レポートを確認できる
  • システム管理者が社員、部署、シフトを管理できる

■ 6. 解決方針

本システムでは、Laravelを中心としたWebアプリケーションを構築し、勤怠管理に必要な基本機能を段階的に実装する。

Browser
   ↓
AWS EC2
   ↓
Docker
   ├── Nginx Container
   ├── PHP-FPM / Laravel Container
   └── Node/Vite Build
   ↓
AWS RDS / MySQL 8

画面はBlade、HTML、CSS、JavaScriptで作成し、バックエンドではPHP-FPM / Laravel 12を使用する。AWS EC2上でDockerを利用し、Nginxコンテナ、PHP-FPM / Laravelコンテナ、Node/Vite Buildを分離して構築する。データベースにはAWS RDS / MySQL 8を使用する。

■ 7. システム概要

HR & Attendance Systemは、社員の勤怠情報と休暇申請を管理するWebシステムである。

社員はログイン後、出勤・退勤打刻、勤怠履歴確認、休暇申請を行う。管理者は申請内容を確認し、承認または却下を行う。システム管理者は社員、部署、シフトなどのマスタデータを管理する。

本システムの導入により、以下の効果が期待できる。

■ 15. リスクと対策
リスク影響対策
要件が増えすぎる開発遅延初期リリースの範囲を明確にし、追加機能はPhase2に分ける。
権限設計が複雑になるセキュリティ不備User、Manager、Adminの権限を要件定義段階で明確化する。
DB設計の変更が発生する実装やテストの手戻りER図とテーブル定義を先に作成し、レビューする。
AWS環境構築で詰まるリリース遅延まずDockerローカル環境を完成させ、AWS構築は段階的に進める。
テスト不足品質低下単体、結合、システム試験の観点を設計段階で準備する。
■ 16. まとめ

本企画では、社員の出退勤、勤怠履歴、休暇申請、管理者承認、社員・部署・シフト管理を行うHR & Attendance Systemを開発する。

Laravel 12、PHP 8.4、Blade、MySQL 8、Docker、Nginx、AWS EC2、AWS RDSを使用し、実務に近い構成で勤怠管理システムを設計・実装する。

本システムにより、勤怠管理業務の効率化、入力ミスの削減、申請状況の可視化、管理作業の負担軽減を目指す。
