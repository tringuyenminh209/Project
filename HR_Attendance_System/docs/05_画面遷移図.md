# 1. 本書の目的

本書は、HR & Attendance Systemにおける画面間の遷移を定義するものである。

本書で定義した画面ID、遷移条件、権限別利用範囲は、画面設計、API設計、テスト仕様書の基準とする。

# 3. 共通画面遷移

```mermaid
flowchart TD
    Start["Start"] --> SCR001["SCR-001 ログイン画面"]
    SCR001 -->|Login Success| SCR002["SCR-002 ダッシュボード"]
    SCR001 -->|Login Failed / E001| SCR001
    SCR002 -->|Logout| SCR001
    SCR002 -->|Session Timeout / E010| SCR001
    SCR003["SCR-003 打刻画面"] -->|Session Timeout / E010| SCR001
    SCR004["SCR-004 勤怠履歴画面"] -->|Session Timeout / E010| SCR001
    SCR005["SCR-005 休暇申請画面"] -->|Session Timeout / E010| SCR001
    SCR006["SCR-006 休暇承認画面"] -->|Session Timeout / E010| SCR001
    SCR007["SCR-007 社員管理画面"] -->|Session Timeout / E010| SCR001
    SCR008["SCR-008 部署管理画面"] -->|Session Timeout / E010| SCR001
    SCR009["SCR-009 シフト管理画面"] -->|Session Timeout / E010| SCR001
    SCR010["SCR-010 レポート画面"] -->|Session Timeout / E010| SCR001
```

# 4. User画面遷移

```mermaid
flowchart TD

    SCR002 -->|打刻| SCR003["SCR-003 打刻画面"]

    SCR005 -->|Validation Error / E003| SCR005
```

Manager（管理者）は、User機能に加えて、休暇承認、勤怠状況確認、レポート閲覧、CSV出力を利用する。

```mermaid
    SCR002 -->|打刻| SCR003["SCR-003 打刻画面"]
    SCR002 -->|勤怠履歴/勤怠状況| SCR004["SCR-004 勤怠履歴画面"]
```

# 8. 例外時画面遷移

| エラーID | 発生画面 | 遷移先 | 処理 |

| E007 | SCR-004 / SCR-010 | 同一画面 | データなしメッセージを表示する。 |
| E008 | SCR-010 | SCR-010 | CSV出力エラーを表示する。 |

# 9. 画面遷移ルール

| ルールID | 内容 |
| --- | --- |
| TR-001 | 未ログインユーザーが業務画面へアクセスした場合、SCR-001へ遷移する。 |
| TR-002 | ログイン成功後は権限に関係なくSCR-002へ遷移する。 |
| TR-003 | SCR-002に表示するメニューはユーザー権限により制御する。 |
| TR-004 | 権限外画面へアクセスした場合、E002を表示しSCR-002へ戻す。 |
| TR-005 | 入力エラー発生時は同一画面に留まり、対象項目のエラーを表示する。 |
| TR-006 | 登録、更新、承認、CSV出力の成功後は完了メッセージを表示する。 |
| TR-007 | セッションタイムアウト発生時はSCR-001へ遷移する。 |
