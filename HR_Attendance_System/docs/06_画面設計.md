# 1. 本書の目的

本書は、HR & Attendance Systemの各画面について、表示項目、入力項目、操作、バリデーション、エラー、画面遷移を定義するものである。

本書は、要件定義書、ユースケース、業務フロー、画面遷移図と整合し、後続のAPI設計、テーブル定義、テスト仕様書の基準とする。

---

# 2. 画面設計方針

| 方針ID | 方針 | 内容 |
| --- | --- | --- |
| UI-001 | Role Based UI | User、Manager、Adminの権限に応じて表示メニューを制御する。 |
| UI-002 | Simple Operation | 打刻、申請、承認など主要操作は少ないクリックで完了できるようにする。 |
| UI-003 | Error Visibility | 入力エラー、権限エラー、システムエラーを画面上で明確に表示する。 |
| UI-004 | Traceability | 各画面はSCR、UC、REQ、BF、Error IDと対応させる。 |
| UI-005 | PC First | 初期リリースではPCブラウザを主対象とする。 |
