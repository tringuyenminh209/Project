# 1. 本書の目的

本書は、HR & Attendance Systemのシステム試験仕様を定義する。

単体試験がController/Service/Model単位、結合試験が画面⇔API⇔DBの連結を確認するのに対し、本書ではシステム全体を**エンドユーザーの視点**で通しで動かし、要件定義書の機能要件・非機能要件・セキュリティ要件・受入条件を満たしていることを確認する。

本書はUAT（18）の直前工程であり、Pass後にUATへ移行する。

# 2. 試験方針

| 方針ID | 方針 | 内容 |
| --- | --- | --- |
| ST-POL-001 | Black Box | 内部実装ではなく、画面操作とAPI応答のみに基づいて検証する。 |
| ST-POL-002 | Requirement Based | 要件定義書の機能要件（REQ）、非機能要件（NFR）、セキュリティ要件（SEC）、受入条件（AC）を基準とする。 |
| ST-POL-003 | Full Coverage | FUNC-001〜FUNC-021の全機能について最低1つの正常系・1つの異常系を実施する。 |
| ST-POL-004 | Realistic Scenario | 実際の運用を想定した複数日・複数ユーザーのシナリオで確認する。 |
| ST-POL-005 | Environment Parity | 本番相当の構成（Docker + Nginx + Laravel + RDS想定のMySQL 8）で実施する。 |
| ST-POL-006 | Non-Regression | 結合試験までに検出・修正した項目（work_hours計算式、leave状態設計）を再確認する。 |
| ST-POL-007 | Sign-off Ready | 本書の結果が要件定義書19章の受入条件（AC-001〜AC-009）を満たすことを最終確認する。 |

| 判定 | 条件 |
| --- | --- |
| Pass | 画面表示、業務結果、非機能指標、セキュリティ、受入条件のすべてが要件どおりである。 |
