# 1. 本書の目的

本書は、HR & Attendance Systemの結合試験仕様を定義する。

単体試験仕様書がController、Service、FormRequest、Model、Policyなどを個別に検証するのに対し、本書では**画面⇔API⇔DBを実際に連結した状態**、および**複数のAPIをまたぐ業務シナリオ**が、要件定義書、業務フロー、画面遷移図、API設計、テーブル定義どおりに一貫して動作することを確認する。

本書は、基本設計書、詳細設計書、単体試験仕様書と整合し、システム試験仕様書・UATの前提となる試験仕様書とする。

# 2. 試験方針

| 方針ID | 方針 | 内容 |
| --- | --- | --- |
| IT-POL-001 | Real DB | モックを使用せず、実際のMySQL 8（結合試験環境）に対して確認する。 |
| IT-POL-002 | Screen to API | 画面操作からAPI呼び出しまでを一連で確認する。 |
| IT-POL-003 | Cross Module | 複数の機能・テーブルをまたぐ処理の整合性を確認する。 |
| IT-POL-004 | Scenario Based | 業務フロー（BF）単位でEnd-to-Endのシナリオを確認する。 |
| IT-POL-005 | Navigation Consistency | 画面遷移図で定義した遷移が実際の画面遷移と一致することを確認する。 |
| IT-POL-006 | Audit Verification | 重要操作がaudit_logsへ実際に記録されることをDBで確認する。 |
| IT-POL-007 | Regression Aware | 単体試験で検出済みの修正（work_hours計算式、leave状態設計など）が結合環境でも正しく反映されていることを確認する。 |

# 6. 判定基準

| 判定 | 条件 |
| --- | --- |
| Pass | 画面操作の結果、API呼び出し、DB反映、画面表示、画面遷移がすべて仕様どおりである。 |
| Fail | いずれかの層（画面／API／DB）で仕様との不一致、データ不整合、遷移不一致がある。 |
| Blocked | 前提となる機能・環境・データが未整備で実行できない。 |

# 7. 共通試験観点

| 観点ID | 観点 | 確認内容 |
| --- | --- | --- |
| IT-COM-001 | Endpoint一致 | 画面操作が呼び出すAPIのMethod/Endpointが画面設計・API設計と一致する。 |
| IT-COM-002 | DB反映 | APIレスポンスの内容が実際にDBへ反映された値と一致する（モックなし）。 |
| IT-COM-003 | Transaction | 複数テーブルを更新する処理で、途中失敗時に不整合データが残らない。 |
| IT-COM-004 | 画面遷移一致 | 成功時・失敗時の遷移先が画面遷移図（05）と一致する。 |
| IT-COM-005 | 権限一貫性 | 権限外操作が画面・API双方で一貫してブロックされる（画面メニュー非表示＋API 403）。 |
| IT-COM-006 | Audit反映 | 重要操作後にaudit_logsへ実際に記録される。 |
| IT-COM-007 | エラー表示一致 | E001〜E010がAPIレスポンスと画面のエラーメッセージ（06章 7.エラーメッセージ）に一致して表示される。 |
| IT-COM-008 | Sensitive Data | password_hashや内部例外詳細が画面・API双方に露出しない。 |

# 18. まとめ

本書では、HR & Attendance Systemの結合試験として、画面⇔API⇔DBの連結確認、モジュール間結合（認証・勤怠・休暇・レポート・マスタ管理・監査）、業務フローを横断するシナリオ試験、画面遷移試験、非機能観点の結合確認を定義した。

特に、単体試験の整合性レビューで判明したwork_hours計算式（BR-ATT-005）、leave_requestsの状態設計（Completedを持たず画面側でend_date経過を判定する方針）、シフト変更時のwork_hoursスナップショット方針については、実データ・実DBを用いた結合試験（IT-ATT-003、IT-ATT-008、IT-LEV-007、IT-SCN-006）で重点的に確認する。

本仕様をもとに、Laravel Feature Test（実DB接続）またはブラウザ結合試験ツールで試験を実施し、Pass後にシステム試験仕様書（17）、UAT（18）へ進む。
