# 1. 本書の目的

本書は、HR & Attendance Systemで使用するMySQL 8向けの物理テーブル定義を定義するものである。

本書はER図の論理設計をもとに、カラム名、データ型、NULL可否、主キー、外部キー、一意制約、インデックスを明確にし、Laravel migration、API設計、実装、テスト仕様書の基準とする。

---

# 2. テーブル設計方針

| 方針ID | 方針 | 内容 |
| --- | --- | --- |
| TBL-001 | MySQL 8 | データベースはAWS RDS / MySQL 8を想定する。 |
| TBL-002 | Laravel Migration | Laravel 12のmigrationで作成しやすい型を使用する。 |
| TBL-003 | Surrogate Key | 各テーブルの主キーは`id`を基本とする。 |
| TBL-004 | Soft Delete Priority | 履歴整合性が必要なマスタは`status`で無効化する。 |
| TBL-005 | Auditability | 重要操作は`audit_logs`に記録する。 |
| TBL-006 | Timestamp | 基本テーブルは`created_at`、`updated_at`を持つ。 |

# 11. まとめ

本書では、HR & Attendance Systemの物理テーブルとして、roles、departments、shifts、employees、attendance_records、leave_requests、audit_logsの7テーブルを定義した。

本書のテーブル定義、外部キー、インデックス、初期データを基準として、Laravel migration、API設計、テスト仕様書へ展開する。
